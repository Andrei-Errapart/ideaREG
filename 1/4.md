## Idea

### Title

**ULi (Universal Link)** — single-wire bidirectional clock-synchronous data transfer protocol with PLL-friendly encoding.

### Description

ULi is an electrical signalling protocol and encoding scheme that allows full-duplex bidirectional data transfer between two nodes over a single electrical wire, using a clock-synchronous frame structure that can be recovered by a standard FPGA PLL.

**Frame structure.** Data is transmitted in 8-clock-period frames. Each frame is divided into two equal halves:

- **Master half** (clock periods 1-4): the master node transmits.
- **Slave half** (clock periods 5-8): the slave node transmits.

Each frame carries one data bit in each direction, with master and slave each transmitting independently of each other's data.

**Master encoding.** The master's 4-clock transmission slot uses two patterns:
- `1000` represents data bit *0*
- `1110` represents data bit *1*

Both master patterns begin with a rising edge at the start of clock period 1. This rising edge is guaranteed to occur in every frame regardless of data content, providing a periodic timing reference for the slave's PLL.

**Slave encoding.** The slave's 4-clock transmission slot uses two patterns:
- `0000` represents data bit *0*
- `0110` represents data bit *1*

Both slave patterns begin with the line at 0 (because the master's transmission has returned the line to 0 by the end of clock period 4) and end with the line at 0 (returning the line to idle before the next frame's master transmission begins).

**Clock recovery.** The slave uses a standard FPGA PLL to recover the clock from the master's rising edge. The PLL locks onto the periodic rising edge at the start of each frame. Once locked, the slave's receive logic blanks the slave's own transmission half of the frame (clock periods 5-8) from the PLL input. This prevents the slave's own transmissions from interfering with the PLL lock, which would otherwise be destabilised by the rising edges in the slave's own data patterns.

The blanking mechanism is what allows ULi to use a standard FPGA PLL rather than requiring custom clock-data-recovery logic. The PLL sees only the master's half of each frame, which has a guaranteed rising edge at a known position, providing the clean periodic signal that PLLs are designed to recover.

### Why this matters

Single-wire bidirectional protocols with these combined properties are uncommon. Most existing single-wire protocols sacrifice at least one of: full bidirectionality, symmetric throughput, clock-synchronous operation, or compatibility with standard clock-recovery hardware. ULi is designed to provide all four.

Application domains where ULi may be useful include:
- Sensor networks where wiring cost matters
- Backplane communication where pin counts matter
- Connector-constrained systems (medical devices, aerospace electronics)
- Inter-chip communication on space-constrained PCBs
- Any system where reducing the wire count from three or four to one provides real cost or space savings

### Domain or field

Digital electronics, embedded systems, FPGA design, communication protocols, signalling and encoding.

### References to related ideas

- **1-Wire** (Dallas Semiconductor / Maxim): single-wire master-slave protocol, but not symmetric or clock-synchronous in the sense ULi is.
- **I2C** (Philips / NXP): two-wire bidirectional protocol; ULi achieves similar bidirectionality with one wire.
- **SPI** (Motorola): synchronous but requires three or four wires.
- **CAN, LIN, RS-485, Ethernet**: all multi-wire protocols.
- **Manchester encoding**: provides guaranteed transitions for clock recovery, similar in spirit to ULi's master-half rising-edge guarantee, but Manchester does not address bidirectionality on a single wire.

ULi's specific contribution is the combination of single-wire physical layer, full-duplex symmetric bidirectionality, clock-synchronous frame structure, and compatibility with standard FPGA PLL hardware through the blanking mechanism.

## Remarks

Initial hardware testing has been completed. The link layer (frame structure, encoding, clock recovery, blanking) has been verified to function as designed. Higher protocol layers (framing of larger data structures, error detection, addressing for multi-node networks) have not been implemented or tested.

The idea was conceived in 2016. It has been carried in private notes for ten years before being registered here on 7 May 2026. The registration establishes a public timestamp for the idea as described above; it does not establish priority for any specific implementation, nor does it preclude others from independently arriving at similar designs.

The codename *ULi* (for *Universal Link*) is intended to be evocative rather than legally distinctive. The name may change if the protocol is developed further toward production use.
